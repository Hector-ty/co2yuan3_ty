# 在手机上运行移动端应用 - 完整指南

## 🎯 前提条件

1. ✅ 电脑和手机连接到**同一个 WiFi 网络**
2. ✅ 移动端应用已在 Docker 中运行
3. ✅ 后端服务正在运行

## 📱 方法一：通过局域网 IP 访问（推荐）

### 步骤 1: 获取电脑的 IP 地址

#### Windows 系统：

```powershell
# 方法1：使用 PowerShell（推荐）
ipconfig | findstr IPv4

# 方法2：查看详细信息
ipconfig

# 查找类似这样的IP地址：
# IPv4 地址 . . . . . . . . . . . . : 192.168.1.100
```

#### Linux/Mac 系统：

```bash
# Linux
ip addr show | grep "inet "

# Mac
ifconfig | grep "inet "
```

**找到类似 `192.168.x.x` 或 `10.x.x.x` 的 IP 地址**

### 步骤 2: 配置 Docker 允许外部访问

默认情况下，Docker 端口已经映射到主机，可以直接访问。如果需要，可以检查端口映射：

```bash
# 查看端口映射
docker ps | findstr mobile
```

### 步骤 3: 配置 Windows 防火墙（重要）

如果手机无法访问，需要允许端口 81 通过防火墙：

#### 方法 A：使用 PowerShell（推荐）

```powershell
# 允许端口 81 通过防火墙
New-NetFirewallRule -DisplayName "Mobile App Port 81" -Direction Inbound -LocalPort 81 -Protocol TCP -Action Allow
```

#### 方法 B：使用图形界面

1. 打开 **Windows Defender 防火墙**
2. 点击 **高级设置**
3. 点击 **入站规则** -> **新建规则**
4. 选择 **端口** -> **下一步**
5. 选择 **TCP**，输入端口 **81**
6. 选择 **允许连接**
7. 应用到所有配置文件
8. 命名规则（如：移动端应用端口 81）

### 步骤 4: 在手机上访问

假设你的电脑 IP 地址是 `192.168.1.100`：

1. **打开手机浏览器**（Chrome、Safari 等）
2. **输入地址**：
   ```
   http://192.168.1.100:81
   ```
3. **访问应用**

### 步骤 5: 测试连接

如果无法访问，检查：

```bash
# 1. 检查移动端容器是否运行
docker ps | findstr mobile

# 2. 检查端口是否监听
netstat -ano | findstr :81

# 3. 测试本地访问
curl http://localhost:81

# 4. 测试局域网访问（从电脑上测试）
curl http://192.168.1.100:81
```

## 🌐 方法二：配置 Nginx 监听所有网络接口

如果方法一不行，可以修改 Nginx 配置以监听所有网络接口。

### 修改 Nginx 配置

需要修改 `MT/nginx.conf`，将 `server_name` 改为 `_` 或 `0.0.0.0`：

```nginx
server {
  listen 81;
  server_name _;  # 监听所有域名
  # 或
  server_name 0.0.0.0;  # 监听所有 IP
  # ...
}
```

然后重新构建：

```bash
docker-compose up -d --build mobile
```

## 📱 方法三：使用 ngrok 内网穿透（外网访问）

如果需要从外网访问（不在同一网络），可以使用 ngrok：

### 步骤 1: 安装 ngrok

访问 https://ngrok.com/ 下载并安装

### 步骤 2: 启动 ngrok

```bash
ngrok http 81
```

### 步骤 3: 使用提供的 URL

ngrok 会提供一个类似 `https://xxxx.ngrok.io` 的 URL，可以在手机上访问。

## 🔧 故障排除

### 问题 1: 手机无法访问

**检查项**：
1. ✅ 电脑和手机在同一 WiFi
2. ✅ 防火墙已开放端口 81
3. ✅ Docker 容器正在运行
4. ✅ IP 地址正确

**解决方法**：
```bash
# 查看移动端日志
docker-compose logs mobile

# 重启移动端服务
docker-compose restart mobile
```

### 问题 2: API 请求失败

**原因**：后端服务未运行或无法访问

**解决方法**：
```bash
# 检查后端服务
docker ps | findstr backend

# 检查后端日志
docker-compose logs backend

# 确保所有服务运行
docker-compose ps
```

### 问题 3: 防火墙阻止

**解决方法**：
- 临时关闭防火墙测试（不推荐）
- 或按照上述步骤配置防火墙规则

### 问题 4: 端口被占用

**解决方法**：
```bash
# 查看端口占用
netstat -ano | findstr :81

# 修改 docker-compose.yml 中的端口映射
# 将 "81:81" 改为 "8081:81"
```

## 💡 小贴士

1. **固定 IP 地址**：可以在路由器中为电脑设置固定 IP，方便后续访问
2. **创建书签**：在手机浏览器中保存应用地址为书签
3. **添加到主屏幕**：大多数手机浏览器支持"添加到主屏幕"，可以像 App 一样使用
