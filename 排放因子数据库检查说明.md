# 排放因子数据库与项目代码一致性检查

## 概述

当您手动修改MongoDB数据库中的排放因子后，可以使用以下方法检查数据库与项目代码之间是否存在问题。

## 检查方法

### 方法1：使用API端点（推荐）

1. 启动服务器
2. 以管理员身份登录
3. 访问API端点：`GET /api/emission-factors/check`
4. 查看返回的JSON报告

### 方法2：使用命令行脚本

```bash
node server/utils/checkEmissionFactors.js
```

## 检查内容

检查脚本会验证以下内容：

### 1. 必需的排放因子是否存在

系统期望以下排放因子存在于数据库中：

**固体燃料 (solid)**:
- `anthracite` (无烟煤)
- `bituminousCoal` (烟煤)
- `lignite` (褐煤)

**液体燃料 (liquid)**:
- `fuelOil` (燃料油)
- `gasoline` (汽油)
- `diesel` (柴油)
- `kerosene` (煤油)
- `lpg` (液化石油气)
- `lng` (液化天然气)

**气体燃料 (gas)**:
- `naturalGas` (天然气)
- `cokeOvenGas` (焦炉煤气)
- `pipelineGas` (管道煤气)

**间接排放 (indirect)**:
- `electricity` (电力)
- `heat` (热力)

### 2. 数据格式验证

- `value` 字段必须是有效数字
- `value` 字段不应为负数
- `unit` 字段必须存在
- `category` 字段必须在允许的枚举值中

### 3. 已弃用的排放因子

检查是否存在已弃用的排放因子：
- 固体燃料：`cokingCoal`, `briquettes`, `coke`, `otherCokingProducts`
- 液体燃料：`crudeOil`, `naphtha`, `asphalt`, `lubricants`, `petroleumCoke`, `petrochemicalFeedstock`, `otherOils`
- 气体燃料：`refineryGas`
- 移动源：整个 `mobile` 类别已弃用

### 4. 名称映射

检查排放因子名称是否能正确映射到规范名称（通过 `canonicalizeFactorName` 函数）

## 系统工作原理

### 排放因子优先级

1. **数据库优先**：计算引擎首先从数据库读取排放因子
2. **默认值回退**：如果数据库中没有某个排放因子，会使用 `server/utils/emissionFactors.js` 中的默认值
3. **覆盖机制**：数据库中的值会覆盖默认值

### 计算流程

```javascript
// 1. 克隆默认值
const factors = cloneDefaultFactors();

// 2. 从数据库读取并覆盖
dbFactors.forEach(factor => {
  factors[category][canonicalName] = factor.value; // 数据库值覆盖默认值
});

// 3. 使用最终值进行计算
breakdown.fossilFuels += (fossilFuels.solid[fuel] || 0) * factor;
```

## 常见问题

### 问题1：修改了数据库但计算仍使用旧值

**原因**：
- 服务器可能缓存了数据
- 数据库中的 `name` 字段与代码期望的名称不匹配

**解决方法**：
1. 重启服务器
2. 检查数据库中的 `name` 字段是否正确（使用检查脚本）
3. 确认 `canonicalizeFactorName` 能正确映射名称

### 问题2：缺少某些排放因子

**影响**：
- 如果数据库中没有某个排放因子，系统会使用默认值
- 如果默认值也不存在，计算会抛出错误

**解决方法**：
1. 使用检查脚本查看缺少哪些排放因子
2. 通过API或管理界面添加缺失的排放因子
3. 或使用 `POST /api/emission-factors/initialize` 初始化所有默认值

### 问题3：存在已弃用的排放因子

**影响**：
- 已弃用的排放因子会被忽略，不会参与计算
- 但会占用数据库空间

**解决方法**：
1. 使用 `POST /api/emission-factors/delete-deprecated` 批量删除
2. 或手动在数据库中删除

### 问题4：单位字段不匹配

**说明**：
- `unit` 字段主要用于显示和记录
- 实际计算使用的是 `value` 字段
- 但建议保持单位字段的准确性

## 建议的检查流程

1. **修改数据库后**：
   ```bash
   # 运行检查脚本
   node server/utils/checkEmissionFactors.js
   ```

2. **查看报告**：
   - 检查是否有 ❌ 问题
   - 查看 ⚠️ 警告
   - 确认 ℹ️ 信息

3. **修复问题**：
   - 根据报告修复数据格式问题
   - 删除已弃用的排放因子
   - 添加缺失的排放因子

4. **验证计算**：
   - 创建测试数据
   - 验证计算结果是否符合预期

## API端点

- `GET /api/emission-factors/check` - 检查数据库一致性（需要管理员权限）
- `POST /api/emission-factors/initialize` - 初始化默认排放因子（需要管理员权限）
- `POST /api/emission-factors/delete-deprecated` - 删除已弃用的排放因子（需要管理员权限）

## 注意事项

1. **备份数据库**：在手动修改数据库前，建议先备份
2. **测试环境**：建议先在测试环境验证修改
3. **重启服务器**：修改数据库后，建议重启服务器以确保缓存更新
4. **验证计算**：修改后应验证计算结果是否正确
