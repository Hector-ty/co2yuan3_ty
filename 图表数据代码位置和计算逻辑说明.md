# 图表数据代码位置和计算逻辑说明

本文档详细说明了项目中所有图表的数据代码位置和计算逻辑。

## 一、图表组件位置汇总

### 1. 客户端（client）图表组件
所有图表组件位于：`client/src/components/charts/`

- **DonutChart.jsx** - 环形图（排放源占比）
- **StackedAreaChart.jsx** - 堆叠面积图（近5年排放源结构变化）
- **DualAxisChart.jsx** - 双轴图（单位建筑面积/人均碳排放）
- **LineChart.jsx** - 折线图（碳排放总量年均变化率）
- **StackedBarChart.jsx** - 堆叠柱状图（年度排放量构成）
- **EmissionMapChart.jsx** - 地图图表（排放地图）
- **EmissionPieChart.jsx** - 饼图（排放构成，用于ChartAnalysis）
- **YearlyEmissionBarChart.jsx** - 年度排放柱状图（年度排放量）
- **AnnualChangeLineChart.jsx** - 年变化率折线图（年均变化率）
- **IntensityBarChart.jsx** - 强度柱状图（排放强度）

### 2. 移动端（MT）图表组件
所有图表组件位于：`MT/src/components/charts/`

- **DonutChart.jsx** - 环形图

## 二、各图表详细说明

### 1. 排放源占比（DonutChart - 环形图）

**组件位置：**
- `client/src/components/charts/DonutChart.jsx`
- `MT/src/components/charts/DonutChart.jsx`

**数据来源：**
- 数据计算位置：`client/src/hooks/useDataScreen.js` (第64-72行)
- 数据字段：`charts.donutData`

**计算逻辑：**
```javascript
// 从最新数据的breakdown中提取各排放源数据
const breakdown = latestEmissions?.breakdown || {};
const { fossilFuels = 0, fugitiveEmissions = 0, electricity = 0, heat = 0 } = breakdown;

const donutData = [
  { value: fossilFuels || 0, name: '化石燃料' },
  { value: fugitiveEmissions || 0, name: '逸散排放' },
  { value: electricity || 0, name: '外购电力' },
  { value: heat || 0, name: '外购热力' },
];
```

**数据说明：**
- **fossilFuels（化石燃料）**：来自后端计算的 `breakdown.fossilFuels`
  - 计算公式：固体燃料 + 液体燃料 + 气体燃料
  - 固体燃料：`燃料消耗量(t) × 排放因子(tCO₂/t)`
  - 液体燃料：`燃料消耗量(t) × 排放因子(tCO₂/t)`
  - 气体燃料：`(燃料消耗量(m³) / 10000) × 排放因子(tCO₂/万Nm³)`
  
- **fugitiveEmissions（逸散排放）**：来自后端计算的 `breakdown.fugitiveEmissions`
  - 计算公式：空调系统逸散排放 + 灭火系统逸散排放
  - 空调系统：`Σ(用户输入值 × 排放因子)`（仅当输入值 ≥ 阈值时计入）
  - 灭火系统：`Σ(用户输入值 × 排放因子)`（仅当输入值 ≥ 阈值时计入）
  
- **electricity（外购电力）**：来自后端计算的 `breakdown.electricity`
  - 计算公式：`净外购电量(10⁴kWh) × 10 × 排放因子(tCO₂/MWh)`
  
- **heat（外购热力）**：来自后端计算的 `breakdown.heat`
  - 计算公式：`净外购热力(GJ) × 排放因子(tCO₂e/GJ)`

**后端计算位置：** `server/utils/calculationEngine.js` (第52-209行)

### 4. 碳排放总量年均变化率（LineChart - 折线图）

**组件位置：**
- `client/src/components/charts/LineChart.jsx`
- `MT/src/components/charts/LineChart.jsx`

**数据来源：**
- 数据计算位置：`client/src/hooks/useDataScreen.js` (第89-99行)
- 数据字段：`charts.lineChartData`

**计算逻辑：**
```javascript
const changeRates = [];
for (let i = 1; i < last5YearsData.length; i++) {
  const prev = last5YearsData[i - 1]?.calculatedEmissions?.totalEmissions || 0;
  const curr = last5YearsData[i]?.calculatedEmissions?.totalEmissions || 0;
  if (prev > 0) {
    changeRates.push(((curr - prev) / prev * 100).toFixed(2));
  } else {
    changeRates.push('0.00');
  }
}
const lineChartData = { years: years.slice(1), changeRates };
```

**数据说明：**
- 计算相邻两年之间的排放总量变化率
- 计算公式：`((当年排放总量 - 前一年排放总量) / 前一年排放总量) × 100%`
- 如果前一年排放总量为0，则变化率设为0.00%
- X轴显示年份（从第二年开始，因为第一年没有变化率）
- Y轴显示变化率百分比

**注意：** 在 `AnnualChangeLineChart.jsx` 中有更详细的处理逻辑，包括：
- 处理前一年为0的情况（设为100%增长）
- 处理当年为0的情况（设为-100%减少）
- 处理两年都为0的情况（设为0%）

### 10. 年均变化率（AnnualChangeLineChart - 年变化率折线图）

**组件位置：**
- `client/src/components/charts/AnnualChangeLineChart.jsx`

**使用位置：**
- `client/src/components/ChartAnalysis.jsx` (第162-166行)

**数据来源：**
- 数据来源：`yearlyDataForSelectedRegion`（同一区域的所有年度数据）

**计算逻辑：**
```javascript
for (let i = 1; i < sortedData.length; i++) {
  const currentYearData = sortedData[i];
  const previousYearData = sortedData[i - 1];
  
  const currentEmissions = Number(currentYearData?.calculatedEmissions?.totalEmissions) || 0;
  const previousEmissions = Number(previousYearData?.calculatedEmissions?.totalEmissions) || 0;

  let changeRate = 0;
  
  // 处理各种边界情况
  if (previousEmissions === 0 && currentEmissions > 0) {
    changeRate = 100; // 从0到有值，设为100%增长
  } else if (previousEmissions > 0 && currentEmissions === 0) {
    changeRate = -100; // 从有值到0，设为-100%减少
  } else if (previousEmissions > 0) {
    changeRate = ((currentEmissions - previousEmissions) / previousEmissions) * 100;
  } else {
    changeRate = 0; // 两年都为0
  }
  
  const finalChangeRate = isFinite(changeRate) ? Number(changeRate.toFixed(2)) : 0;
}
```

**数据说明：**
- 计算相邻两年之间的排放总量变化率
- 计算公式：`((当年排放总量 - 前一年排放总量) / 前一年排放总量) × 100%`
- 特殊处理：
  - 前一年为0，当年不为0：变化率 = 100%（表示从无到有）
  - 前一年不为0，当年为0：变化率 = -100%（表示完全减少）
  - 两年都为0：变化率 = 0%
- 显示平滑曲线，包含最大值和最小值标记点
- 不支持对比模式（需要所有对比单位连续两年的数据）

## 三、后端计算引擎说明

### 计算引擎位置
`server/utils/calculationEngine.js`

### 主要计算函数
`calculateEmissions(activityData)` - 计算排放量

### 计算流程

1. **获取排放因子**
   - 从数据库获取最新排放因子
   - 合并默认排放因子

2. **直接排放计算**
   - **化石燃料燃烧**（breakdown.fossilFuels）
     - 固体燃料：`燃料消耗量(t) × 排放因子(tCO₂/t)`
     - 液体燃料：`燃料消耗量(t) × 排放因子(tCO₂/t)`
     - 气体燃料：`(燃料消耗量(m³) / 10000) × 排放因子(tCO₂/万Nm³)`
   
   - **逸散排放**（breakdown.fugitiveEmissions）
     - 空调系统：`Σ(用户输入值 × 排放因子)`（仅当输入值 ≥ 阈值时计入）
     - 灭火系统：`Σ(用户输入值 × 排放因子)`（仅当输入值 ≥ 阈值时计入）
     - 总逸散排放 = 空调系统 + 灭火系统

3. **间接排放计算**
   - **外购电力**（breakdown.electricity）
     - `净外购电量(10⁴kWh) × 10 × 排放因子(tCO₂/MWh)`
   
   - **外购热力**（breakdown.heat）
     - `净外购热力(GJ) × 排放因子(tCO₂e/GJ)`

4. **总量计算**
   - 直接排放总值 = 化石燃料 + 逸散排放
   - 间接排放总值 = 外购电力 + 外购热力
   - 总排放量 = 直接排放量 + 间接排放量

5. **强度指标计算**
   - 单位建筑面积碳排放量 = `总排放量 / 机关单位建筑面积`（kgCO₂/m²）
   - 人均碳排放量 = `总排放量 / 机关人员数量`（tCO₂/人）

### 返回数据结构
```javascript
{
  breakdown: {
    fossilFuels: 0,        // 化石燃料排放
    fugitiveEmissions: 0,  // 逸散排放
    airConditioning: 0,    // 空调系统逸散排放
    fireSuppression: 0,    // 灭火系统逸散排放
    electricity: 0,        // 外购电力排放
    heat: 0               // 外购热力排放
  },
  totalDirect: 0,          // 直接排放总值
  totalIndirect: 0,       // 间接排放总值
  totalEmissions: 0,      // 总排放量
  emissionIntensityByArea: 0,      // 单位建筑面积碳排放
  emissionIntensityByPerson: 0     // 人均碳排放
}
```

## 六、注意事项

1. **数据过滤**：所有图表数据都会过滤掉 `calculatedEmissions` 为空的数据
2. **数据排序**：按年份倒序排列，最新数据在前
3. **空值处理**：所有计算都使用 `|| 0` 处理空值情况
4. **单位换算**：
   - 气体燃料需要除以10000进行单位换算
   - 电力排放需要乘以10进行单位换算
5. **阈值处理**：逸散排放中，如果输入值小于阈值，该部分排放不计入
6. **对比数据**：对比模式下，同级别单位平均值通过求和后除以单位数量计算

## 八、移动端（MT）说明

移动端的图表组件与客户端基本相同，主要区别：
1. 图表样式适配移动端（字体大小、间距等）
2. 通过 `ChartDetailPage` 单独显示每个图表
3. 数据来源和计算逻辑与客户端完全一致
