# MongoDB 用户数据完整性检查说明

## 概述

本工具用于检查 MongoDB 数据库中每个用户是否包含所有必需的字段。

### 需要检查的字段

1. **邮箱** (email) - 必填
2. **密码** (password) - 必填
3. **单位名称** (unitName) - 必填
4. **统一社会信用代码** (creditCode) - 必填
5. **详细地址** (address) - 必填
6. **建筑面积(平方米)** (buildingArea) - 必填，数字类型
7. **用能人数(人)** (personnelCount) - 必填，数字类型
8. **填报联系人** (contactPerson) - 必填
9. **联系电话** (contactPhone) - 必填

---

## 使用方法

### 方法 1：使用 PowerShell 脚本（推荐）

#### 检查数据

在项目根目录运行：

```powershell
.\check-user-data.ps1
```

#### 修复数据（为缺失字段设置默认值）

在项目根目录运行：

```powershell
.\fix-user-data.ps1
```

**注意**：修复脚本会为缺失字段设置默认值，建议先运行检查脚本查看问题。

### 方法 2：直接运行 Node.js 脚本

#### 检查数据

```bash
# 设置 MongoDB URI（如果未在环境变量中设置）
$env:MONGODB_URI = "mongodb://localhost:27017/carbon_platform"

# 运行检查脚本
node server/scripts/checkUserData.js
```

#### 修复数据

```bash
# 设置 MongoDB URI
$env:MONGODB_URI = "mongodb://localhost:27017/carbon_platform"

# 运行修复脚本
node server/scripts/fixUserData.js
```

### 方法 3：在 Docker 容器中运行

如果使用 Docker，可以在容器中运行：

```bash
# 进入后端容器
docker exec -it <backend-container-name> sh

# 设置环境变量（如果需要）
export MONGODB_URI="mongodb://mongodb:27017/carbon_platform"

# 运行检查脚本
node scripts/checkUserData.js
```

## 修复脚本说明

### 默认值设置

修复脚本会为缺失字段设置以下默认值：

| 字段 | 默认值 |
|------|--------|
| 统一社会信用代码 | `000000000000000000` |
| 详细地址 | `未填写` |
| 建筑面积 | `0` |
| 用能人数 | `0` |
| 填报联系人 | `未填写` |
| 联系电话 | `00000000000` |

### 注意事项

1. **密码字段**：修复脚本不会修改密码字段，如果密码缺失，需要用户重新设置
2. **邮箱和单位名称**：这些是核心字段，如果缺失可能需要删除用户或联系管理员
3. **默认值**：修复后的默认值需要用户手动更新为真实数据

## 常见问题

### Q: 如何设置 MONGODB_URI？

A: 可以通过以下方式设置：

1. **环境变量**：
   ```powershell
   $env:MONGODB_URI = "mongodb://localhost:27017/carbon_platform"
   ```

2. **Docker Compose**：
   如果使用 Docker，MONGODB_URI 通常在 `docker-compose.yml` 中设置

3. **直接修改脚本**：
   在脚本开头添加：
   ```javascript
   process.env.MONGODB_URI = "mongodb://localhost:27017/carbon_platform";
   ```

### Q: 检查脚本显示某些字段为 null 或 undefined，怎么办？

A: 这些是旧数据，可能是在添加新字段之前注册的用户。可以：
1. 运行修复脚本设置默认值
2. 让用户重新登录并更新信息
3. 手动在数据库中更新

### Q: 如何查看特定用户的数据？

A: 可以修改检查脚本，添加用户过滤：

```javascript
// 只检查特定邮箱的用户
const users = await Account.find({ email: 'user@example.com' }).select('+password');
```

### Q: 修复脚本会修改哪些字段？

A: 修复脚本只会修复以下字段（如果缺失）：
- 统一社会信用代码
- 详细地址
- 建筑面积
- 用能人数
- 填报联系人
- 联系电话

**不会修改**：
- 邮箱
- 密码
- 单位名称
- 地区
- 角色

## 建议的工作流程

1. **首次检查**：
   ```powershell
   .\check-user-data.ps1
   ```

2. **查看报告**：了解哪些用户数据不完整

3. **决定修复策略**：
   - 如果问题较少：手动联系用户更新
   - 如果问题较多：运行修复脚本设置默认值

4. **运行修复**（如需要）：
   ```powershell
   .\fix-user-data.ps1
   ```

5. **再次检查**：确认修复结果
   ```powershell
   .\check-user-data.ps1
   ```

6. **通知用户**：提醒用户更新默认值为真实数据
