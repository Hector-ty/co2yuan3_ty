# 日志文件说明和示例

## 📝 什么是日志文件？

**日志文件**是程序运行时自动记录的信息文件，类似于"日记"。它记录了：
- 程序启动和关闭信息
- 错误和异常信息
- 用户操作记录
- 系统运行状态
- 调试信息

在 Docker 容器中，所有输出到**标准输出（stdout）**和**标准错误（stderr）**的内容都会被 Docker 自动捕获并保存到日志文件中。

---

## 📍 Docker 日志文件的位置

### Windows Docker Desktop
```
C:\Users\<用户名>\AppData\Local\Docker\log-driver\
```

### Linux
```
/var/lib/docker/containers/<容器ID>/<容器ID>-json.log
```

每个容器都有自己独立的日志文件，文件名通常是：`<容器ID>-json.log`

---

## 🔍 如何查看容器日志

### 方法 1：使用 Docker 命令（推荐）

```bash
# 查看容器的最新日志
docker logs <容器名或ID>

# 查看最后 100 行日志
docker logs --tail 100 <容器名>

# 实时跟踪日志（类似 tail -f）
docker logs -f <容器名>

# 查看带时间戳的日志
docker logs -t <容器名>

# 查看最近 1 小时的日志
docker logs --since 1h <容器名>
```

### 方法 2：查看日志文件位置

```bash
# 获取容器的日志文件路径
docker inspect --format='{{.LogPath}}' <容器名>
```

## 📄 日志文件格式示例

Docker 容器的日志文件使用 **JSON 格式**，每行代表一条日志记录。格式如下：

```json
{"log":"2024-01-15 10:30:45 服务器启动成功\n","stream":"stdout","time":"2024-01-15T10:30:45.123456789Z"}
{"log":"用户 root@root.com 登录成功\n","stream":"stdout","time":"2024-01-15T10:31:12.987654321Z"}
{"log":"错误: 数据库连接失败\n","stream":"stderr","time":"2024-01-15T10:32:05.456789123Z"}
```

每条日志包含三个字段：
- `log`: 日志内容
- `stream`: 输出流类型（stdout 或 stderr）
- `time`: 时间戳（ISO 8601 格式）

## 📊 日志文件大小示例

### 典型的日志文件大小

| 服务 | 日志产生速度 | 1小时大小 | 1天大小 |
|------|------------|----------|---------|
| frontend | 很少 | ~1MB | ~10MB |
| mobile | 很少 | ~1MB | ~10MB |
| backend | 中等 | ~5MB | ~50MB |
| ai_service | 中等 | ~10MB | ~100MB |
| mongodb | 很少 | ~2MB | ~20MB |
| ollama | 较多 | ~20MB | ~200MB |

### 没有日志限制的情况

如果**没有配置日志限制**，日志文件会无限增长：
- 运行 1 个月后，单个容器日志可能达到 **1-10GB**
- 运行 1 年后，可能达到 **10-100GB**
- 最终可能占满整个磁盘空间，导致服务器崩溃！

### 配置日志限制后

根据您当前的配置：
- **backend**: 最多保留 5 个文件 × 50MB = **250MB**
- **ollama**: 最多保留 5 个文件 × 100MB = **500MB**
- **frontend**: 最多保留 3 个文件 × 10MB = **30MB**

当日志文件达到最大大小（如 50MB）时，Docker 会自动：
1. 将当前日志文件重命名为 `容器ID-json.log.1`
2. 创建新的日志文件 `容器ID-json.log`
3. 删除最旧的日志文件（如 `.log.5`）

这样就能自动管理日志大小！

## 🔧 实际操作示例

### 1. 查看后端容器的日志

```bash
# 查看所有日志
docker logs co2yuan3-main-backend-1

# 查看最后 50 行
docker logs --tail 50 co2yuan3-main-backend-1

# 实时跟踪日志
docker logs -f co2yuan3-main-backend-1

# 查看今天的日志
docker logs --since 24h co2yuan3-main-backend-1
```

### 2. 查找错误日志

```bash
# 查看包含 "error" 的日志
docker logs co2yuan3-main-backend-1 2>&1 | grep -i error

# 查看包含 "Error" 的日志（前 20 行）
docker logs co2yuan3-main-backend-1 2>&1 | grep -i error | head -20
```

### 3. 查看日志文件的实际位置和大小

```bash
# 获取日志文件路径
docker inspect --format='{{.LogPath}}' co2yuan3-main-backend-1

# 查看日志文件大小（Linux）
du -h /var/lib/docker/containers/*/*-json.log

# 查看日志文件大小（Windows PowerShell）
Get-ChildItem "$env:USERPROFILE\AppData\Local\Docker\log-driver\" -Recurse | 
    Measure-Object -Property Length -Sum | 
    Select-Object @{Name="Size(GB)";Expression={[math]::Round($_.Sum / 1GB, 2)}}
```

## 💡 日志文件的实际用途

### 1. **问题排查**
当应用出现问题时，查看日志可以快速定位原因：
```bash
# 查看最近的错误
docker logs co2yuan3-main-backend-1 2>&1 | grep -i error | tail -20
```

### 2. **监控系统运行状态**
了解系统是否正常运行：
```bash
# 查看启动日志
docker logs co2yuan3-main-backend-1 | grep -i "启动\|ready\|started"
```

### 3. **审计和追踪**
记录用户操作和系统事件：
```bash
# 查看登录日志
docker logs co2yuan3-main-backend-1 | grep -i "login\|登录"
```

### 4. **性能分析**
分析系统性能瓶颈：
```bash
# 查看慢查询日志
docker logs co2yuan3-main-backend-1 | grep -i "slow\|慢"
```

## 📝 总结

1. **日志文件是什么**：记录程序运行信息的文件，存储在 Docker 指定的位置
2. **格式**：JSON 格式，每行一条记录，包含日志内容、流类型和时间戳
3. **大小**：如果不限制，会无限增长，可能占用大量磁盘空间
4. **管理**：通过 `docker-compose.yml` 配置日志轮转，自动限制大小和数量
5. **查看**：使用 `docker logs` 命令查看，无需直接访问文件

**关键点**：日志限制配置**不会影响**应用运行，只是自动清理旧的日志文件，保留最新的日志供您使用！
